Page 1: Dispatch Information Requirements
Critical Notice: No sections can be modified after chart submission unless returned during the QA process. Ensure accuracy before proceeding to patient documentation sections.

Required Dispatch Information

Essential Page 1 Elements

  • Complete dispatch information including all relevant details
  • Complete dispatch number verified for accuracy
  • Incident type and priority correctly documented
  • Separate chart requirement - each patient needs individual chart
  • Patient employer information - V2X, subcontractor name, or military branch
  • Correct Base assignment and location
  • Correct unit number (e.g., Medic 2)
  • Accurate dispatch classification and disposition
  • All crew members listed with roles and transport mode

Dispatch Classification Standards

Call Type Proper Classification Documentation Requirements Special Notes
Emergency Calls Use actual dispatch terminology Exact dispatch reason as received Do NOT use "Adult/Pediatric Medical/Trauma/Neuro"
All Transfers Transfer/Interfacility/Palliative Care Regardless of medical issues Universal classification for all transfers
Standby Events Standby - [Event Type] Specific event description Include event duration and location

Response Mode Documentation

Transport Mode Classification

Emergency Calls (911)

  • Classification: Urgent
  • Transport Code: Code 3 (Lights and Sirens)
  • Documentation: Response mode is emergent
  • Special Considerations: Time-critical interventions

Transfers

  • Classification: Urgent or Priority
  • Transport Code: Code 2 (No lights/sirens unless traffic dictates)
  • Documentation: Interfacility transport requirements
  • Special Considerations: Facility-to-facility protocols

Standby & Pre-scheduled Events

  • Classification: Routine
  • Transport Code: As clinically indicated
  • Documentation: Event coverage details
  • Special Considerations: Preventive care positioning

Personnel & Facility Information

Referring Personnel Requirements

Military Referral Documentation

Must have referring person with complete information:

  • Rank: What they are (example: CPT, SGT, LT)
  • Role: Their position or function
  • Officer Requirement: Must be an officer if military
  • Contact Information: Phone number or radio call sign

Crew Member Documentation

  • All crew members listed with complete names
  • Position designation for each crew member
  • Certification levels (EMT, Paramedic, RN, etc.)
  • Role assignments (Primary, Secondary, Driver, etc.)

Receiving Facility Information

Complete Receiving Facility Documentation

Required Facility & Handoff Information

  • Facility name and location (e.g., Camp Arifjan - US Military Hospital Kuwait)
  • Receiver's name and rank (e.g., Lt Knutson)
  • Officer Requirement: Must be an officer if military facility
  • Role designation (e.g., receiving nurse, ED physician)
  • Contact information and notification requirements
  • Transport mode: Emergent or non-emergent
  • Handoff time: Document exact time of patient transfer
Handoff Documentation: The hand-off report must include who the patient was transferred to, including their name, rank (if military), and role. This ensures continuity of care and proper documentation of patient transfer.

Patient Information & Demographics

Patient Classification Requirements

Mandatory Patient Classification

Required to specify patient category:

  • CTR: Contractor personnel
  • DoD: Department of Defense civilian
  • Mil: Military personnel
  • CIV: Civilian (non-affiliated)
Military Rank Reference: When documenting military personnel, use the correct rank abbreviation to ensure accurate identification and proper protocol adherence. See Appendix A: Military Ranks & Abbreviations for complete reference tables covering all U.S. military branches (Army, Navy, Air Force, Marines, Coast Guard) with both enlisted and officer ranks.

Address & Location Documentation

Address Documentation Protocol: Follow standardized address formats based on location type to ensure consistency and operational security requirements.

Address Format Requirements

On Base Addresses

  • City Field: Use base name (e.g., Arifjan, Buehring)
  • Address Format: Zone, Street, Building, or Location
  • Country: Kuwait or Qatar
  • Standardization: Use consistent formatting

Off Base Addresses

  • Street Information: Include street, road, or highway
  • Reference Points: Cross street, landmark, or identifiable location
  • Country: Kuwait or Qatar
  • GPS Coordinates: When available for remote locations

Address Documentation Standards

  • Common Address Database: Check for frequent locations in system
  • Multiple Units: Same incident requires identical address for all responding units
  • Address Unknown: Check as unknown if security protocols require
  • State Field: Leave blank for international operations
  • Country Field: Where the call was conducted
  • Citizen Field: Patient's country of origin/citizenship
  • Social Security Number: Enter SSN if available

Essential Demographics

Required Patient Information

Basic Demographics

  • Date of Birth: Required on all patients
  • Age: Auto-calculated from DOB (verify accuracy)
  • Gender: Male, Female, Other, Unknown
  • Weight: Required on all patient encounters
  • Height: Needed for medication calculations
  • Patient's Employer: Complete as necessary

Medical History

  • Current Medical History: Including source of information
  • Information Source: Patient, family, medical records
  • Reliability Assessment: Patient's ability to provide history
  • Language Barriers: Interpreter requirements

Medications & Allergies

  • Current Medications: If any, include dosages
  • Medication Source: Patient, family, pill bottles
  • Current Allergies: If any, include reactions
  • Allergy Severity: Mild, moderate, severe reactions

Patient Search & Verification

Search Strategy: Always search for existing patients before creating new records. Duplicate patient records can cause documentation issues and fragment medical history.

Patient Identification Methods

Identity Verification: When searching, verify patient identity using at least two data points (name + DOB, or name + military ID) before selecting existing records. Incorrect patient selection can result in HIPAA violations and documentation errors.
  • Primary Verification: Full name match (allow for nicknames/variations)
  • Secondary Verification: Date of birth within reasonable margin
  • Military Verification: Rank, unit, or service number confirmation
  • Photo Identification: CAC, driver's license, or military ID when possible
  • Medical History Review: Confirm known conditions match patient presentation

Page 2 Essential Elements

  • Impression/Diagnosis area must be complete
  • Chief Complaint: What they called us for, including duration
  • ALS Assessment documentation: Whether performed and why
  • History of Present Illness: Brief synopsis of the event
  • Reason for encounter: Injury or non-injury classification
  • Drug/Alcohol involvement: If applicable
  • Scene description: When applicable to patient condition

Chief Complaint Documentation Standards

Documentation Tip: Use the patient's own words when possible for chief complaint. This provides valuable context and supports medical necessity for transport and treatment decisions. Include the duration and associated symptoms.
Component Character Limit Best Practice Examples
Primary Complaint 50 characters Patient's exact words in quotes "Crushing chest pain," "Can't breathe," "Left foot pain"
Duration Included in HPI Specific timeframe when possible "Started 2 hours ago," "Since this morning"
Associated Symptoms In HPI section Related symptoms (pain, numbness, etc.) "With numbness and tingling"
ALS Assessment Separate field Document if performed and rationale "ALS assessment for chest pain," "BLS appropriate"

History of Present Illness (HPI) Requirements

HPI Documentation Must Include

  • Events leading to injury/illness
  • Mechanism of injury (e.g., kick to foot, fall from height)
  • Timeline of events from initial incident to EMS arrival
  • Previous medical conditions relevant to current complaint
  • Current medications and allergies
  • Pain assessment including location, quality, and severity

Exposure Documentation

Exposure Protocol: Document exposure only if you were actually exposed to something such as a TB patient, needle stick, or infectious disease. Simply arriving on scene does not constitute exposure.

Patient Belongings Documentation

  • List all patient belongings including Common Access Cards (CAC)
  • Document to whom belongings were returned
  • Include valuable items (wallet, jewelry, electronics)
  • Note any items left at facility and responsible party

Factors Affecting Care

This section allows documentation of any factors that affected patient care delivery:

  • Access difficulties (narrow stairs, elevator issues)
  • Weather conditions affecting response or treatment
  • Language barriers requiring interpreters
  • Family interference or assistance
  • Equipment malfunctions or limitations
  • Scene safety concerns

Page 3: Neurological & Airway Assessment

Important Warning: Be careful on this page as most fields have default listings. Only document what you actually assessed. Do not mark assessments as complete if they were not performed.

Standard Neurological Assessment

Neurological Documentation Requirements

Standard neurological assessment includes:

  • LOC (Level of Consciousness)
  • Mental Status
  • Neurological Assessment
  • Any Loss of Consciousness
  • Glasgow Coma Scale (GCS)

Glasgow Coma Scale Documentation

Component Score Range Assessment Criteria Documentation Notes
Eye Opening 1-4 points Spontaneous (4), to voice (3), to pain (2), none (1) Note if eyes swollen shut
Verbal Response 1-5 points Oriented (5), confused (4), inappropriate (3), incomprehensible (2), none (1) Consider intubation, language barriers
Motor Response 1-6 points Obeys commands (6), localizes (5), withdraws (4), flexion (3), extension (2), none (1) Test both sides, note asymmetry

Airway Assessment

Airway Documentation Standard: Airway is patent unless it is compromised or intubated.

  • Patent - Normal, unobstructed airway
  • Compromised - Partial obstruction or difficulty
  • Intubated - Endotracheal tube or supraglottic airway
  • Surgical - Tracheostomy or cricothyrotomy

Pupil & Motor/Sensory Assessment

Required Documentation: Pupils and motor/sensory function must be assessed and documented.

Pupil Assessment

  • Size: Measured in millimeters (1-8mm)
  • Reactivity: Brisk, sluggish, or fixed
  • Equality: Equal or unequal
  • Shape: Round, irregular, or surgical

Motor/Sensory Function

  • Motor Strength: 5/5 scale for each extremity
  • Sensation: Intact, diminished, or absent
  • Reflexes: Present, diminished, or absent
  • Coordination: Intact or impaired

Page 4: Respiratory & Cardiovascular Assessment

Respiratory Assessment Requirements

  • Respiratory effort must be documented
  • Additional documentation required if:
    • Prior oxygen administration
    • Airway device in use (ventilator, ambu bag, etc.)
    • Respiratory distress present
  • Breath sounds required on all patients

Cardiovascular Assessment Requirements

Pulse Assessment Standard: Pulses must be palpated to determine quality. A cardiac monitor can only show rate - manual pulse assessment determines quality, regularity, and strength.
  • Pulse quality assessment through palpation
  • JVD (Jugular Venous Distention) as applicable
  • Edema assessment as applicable
  • Capillary refill on all patients
  • Pacemaker documentation only if patient has one or was applied prior to arrival

Page 5: Secondary Survey

Assessment Integrity: Only document what you actually assess. Do not mark assessments as done if they don't pertain or weren't performed. There is a "not done" dropdown option available.

Secondary Survey Guidelines

Clinical Example: If treating a twisted ankle with no other system involvement and you did not perform a full head-to-toe assessment, do not mark everything as done simply to avoid getting a flag. Document only what was actually assessed.

Physical Assessment Documentation Standards

Required Physical Exam Elements

  • Document all positive findings: Tenderness, bruising, swelling, deformity
  • Vascular assessment: Pedal pulses, capillary refill, skin color/temperature
  • Neurological function: Sensation, movement, strength testing
  • Impression/Diagnosis: Must be clearly documented (e.g., traumatic injury, soft tissue injury)
  • Working diagnosis: Appropriate to findings (e.g., possible metatarsal fracture)

Assessment Categories

Head/Neck Assessment

  • Normocephalic, atraumatic
  • Deformity, contusions, abrasions
  • Lacerations, swelling
  • Penetrating injury
  • Neck rigidity, JVD

Chest Assessment

  • Clear bilateral breath sounds
  • Wheezes, rales, rhonchi
  • Diminished/absent sounds
  • Chest wall deformity
  • Subcutaneous emphysema

Abdomen Assessment

  • Soft, non-tender
  • Rigid, distended
  • Tenderness, guarding
  • Mass, organomegaly
  • Bowel sounds present/absent

Extremities Assessment

  • No deformity, full ROM
  • Fracture, dislocation
  • Edema, cyanosis
  • Pulse, motor, sensation
  • Temperature, capillary refill

Protocol & Dropdown Documentation

Protocol Selection

If a protocol relates to this patient, use the dropdown box to list the protocol. This helps with QA metrics and quality improvement.

Dropdown Box Usage

QA Enhancement: Use all dropdown boxes that are appropriate to the call. This data helps with pulling QA metrics and improving system performance.

Vital Signs Method Avoidance

International SOS Narrative Documentation Policy
Policy: All EMS personnel must use either LCHART or DRAATT narrative formats for every patient encounter. This policy applies to all EMTs, Paramedics, and Captains.

Policy Overview

International SOS EMS Narrative Policy Requirements

  • Format Selection: Choose either LCHART or DRAATT based on case complexity
  • Implementation Date: All documentation must follow approved formats beginning March 1, 2025
  • Quality Assurance: Supervisors will conduct reviews to identify patterns or deviations
  • Compliance Feedback: Quality assurance feedback provided as part of ongoing compliance efforts

Approved Narrative Formats

LCHART Method

Best for: Clinically complex cases requiring detailed assessment and treatment documentation

  • L - Location
  • C - Chief Complaint
  • H - History
  • A - Assessment
  • R - Rx (Treatment)
  • T - Transport

Focus: Patient-centered; emphasizes clinical findings and interventions

DRAATT Method

Best for: Straightforward cases with clear timeline of events

  • D - Dispatch
  • R - Response
  • A - Assessment
  • A - Actions
  • T - Transport
  • T - Transfer

Focus: Chronological flow; focuses on event progression and actions taken

LCHART Method - Detailed Breakdown

When to Use LCHART

  • Cases involve complex clinical assessments (chest pain, trauma, stroke)
  • Detailed documentation of treatments and patient responses is critical
  • Reports need to prioritize clinical accuracy for medical or legal reviews

LCHART Components

L - Location

Purpose: Document where the incident occurred and where the patient was found

  • Exact address or descriptive location
  • Scene details: indoors/outdoors, lighting conditions, hazards
  • Context essential for legal and operational tracking

Example: "Dispatched to 450 Elm St, patient found in the bedroom on the second floor."

C - Chief Complaint

Purpose: Primary reason for EMS call as stated by patient or bystanders

  • Patient's own words when possible (use quotes)
  • Observations when patient unable to communicate
  • Bystander accounts if patient unconscious

Example: "Patient states, 'I feel like I can't breathe.'"

H - History

Purpose: Detailed account of events leading to current situation and relevant medical history

  • Onset: When symptoms started, patient activity
  • Pertinent medical history: Chronic conditions, medications, allergies
  • Events leading to EMS arrival: Witness accounts, mechanisms

Example: "Patient states chest pain began 30 minutes ago while gardening. History of hypertension and high cholesterol. No known allergies."

A - Assessment

Purpose: Documentation of findings from patient assessment

  • Primary survey: Airway, breathing, circulation
  • Secondary survey: Focused exam based on chief complaint
  • Vital signs: Initial and subsequent readings
  • Diagnostic findings: ECG, blood glucose, SpO2

Example: "Patient alert and oriented x3, breathing labored, BP 160/90, HR 112, SpO2 89% on room air. Chest pain 8/10, radiating to left arm."

R - Rx (Treatment)

Purpose: Record of all treatments administered during EMS encounter

  • Medications: Name, dose, route, time, patient response
  • Procedures: IV insertion, oxygen therapy, splinting
  • Changes in patient condition post-treatment

Example: "Administered 324mg ASA PO, 0.4mg Nitro SL x2 with chest pain improvement to 4/10. Oxygen 15L NRB, SpO2 improved to 96%."

T - Transport

Purpose: Details of transport phase, patient condition, and destination

  • Mode of transport: Emergent, non-emergent
  • Destination: Name and type of receiving facility
  • Patient condition en route
  • Handoff report: Information shared with receiving staff

Example: "Transported non-emergently to General Hospital ED. Patient remained stable en route. Report given to RN Smith at 14:35."

DRAATT Method - Detailed Breakdown

When to Use DRAATT

  • Cases are routine or time-sensitive (non-critical transport, basic life support)
  • Clear chronological sequence of events is required
  • Training new EMS providers on fundamental documentation skills

DRAATT Components

D - Dispatch

Purpose: Details about initial call and information received during dispatch

  • Nature of call (e.g., "Unresponsive adult male")
  • Time of dispatch and responding units
  • Pertinent dispatch information: caller description, location hazards

Example: "Dispatched at 13:45 for 52-year-old male, unresponsive. Caller reported patient collapsed at home."

R - Response

Purpose: Documentation of EMS unit's journey to scene and arrival details

  • Time en route, arrival time, scene description
  • Environmental factors: weather, traffic
  • Initial impressions upon arrival

Example: "Unit responded non-emergently, arriving on scene at 13:55. Patient found lying supine on living room floor. No apparent hazards."

A - Assessment

Purpose: Findings from primary and secondary assessments

  • Primary survey: Airway, breathing, circulation (ABC)
  • Secondary survey: Detailed physical exam, focused assessments
  • Vital signs and diagnostic findings

Example: "Primary survey: Airway patent, breathing labored, pulse weak. BP 88/60, HR 120, SpO2 88% on room air. Cyanosis noted, diminished breath sounds bilaterally."

A - Actions

Purpose: All treatments and interventions performed during call

  • Medications administered: name, dose, route, time
  • Procedures performed: CPR, intubation, splinting
  • Patient response to interventions

Example: "Administered 500mL NS IV bolus, BP improved to 102/70. Oxygen therapy initiated at 15L NRB, SpO2 increased to 95%."

T - Transport

Purpose: Details of patient condition and care during transport

  • Mode of transport: emergent, non-emergent
  • Patient condition en route
  • Interventions performed during transport

Example: "Transported emergently to Regional Medical Center. Patient remained stable en route with SpO2 maintained at 95% on 15L NRB."

T - Transfer

Purpose: Handover of care to receiving facility

  • Time of transfer
  • Report given to receiving personnel (name and role)
  • Patient condition upon transfer

Example: "Transferred care to ED RN Johnson at 14:20. Patient stable, SpO2 95% on 15L NRB. Verbal and written report provided."

Method Comparison & Selection Guide

Criteria LCHART DRAATT
Clarity Detailed and logical structure Simple chronological flow
Best Use Complex clinical cases Routine or time-sensitive cases
Learning Curve Robust for advanced providers Easier for beginners
Legal Protection Comprehensive clinical coverage Strong timeline focus
Adaptability Suitable for diverse clinical cases Best for straightforward scenarios

Common Documentation Pitfalls

Avoid These Common Errors:
  • Omitting Key Details: Include pertinent negatives when appropriate to clarify clinical findings (e.g., "No history of diabetes or heart disease")
  • Using Ambiguous Language: Be specific (e.g., "Patient's pain improved from 8/10 to 4/10" vs. "Patient felt better")
  • Inconsistent Terminology: Use approved medical abbreviations, avoid slang
  • Incomplete Vital Signs: Record all vital signs at appropriate intervals
  • Timeline Inconsistencies: Ensure times match across all sections

Best Practices for Both Methods

  • Plan Before Writing: Jot down key points during the call
  • Follow Structure: Document each section in order for consistency
  • Review and Revise: Check narrative for clarity and completeness
  • Practice Regularly: Use mock scenarios to build proficiency
  • Professional Language: Use proper grammar, spelling, and medical terminology
  • Objective Documentation: Be factual, avoid subjective opinions
Quality Documentation Standard: Both LCHART and DRAATT methods ensure comprehensive documentation that meets International SOS standards for patient care, legal compliance, and operational excellence.

QA Workflow & Flag Response Requirements
Mandatory QA Response Policy: All QA flags must be answered no later than 1400 (2:00 PM) the next day if on shift. If departing shift, ALL QA flags must be answered before leaving. Failure to meet these deadlines may result in disciplinary action.

QA Level Progression Overview

QAQuality Assurance - systematic review process to ensure documentation accuracy and clinical appropriateness Philosophy: The Quality Assurance system ensures documentation accuracy, clinical appropriateness, and continuous improvement through structured chart review and feedback processes.

Chart Advancement Process

  1. S0 - Crew Level (Initial Entry)
    • Chart creation and completion by field crew
    • All required documentation must be complete
    • Chart submitted for QA review by clicking "Advance Chart"
    • Chart moves to next defined QA level
  2. QA Level Progression
    • Chart advances through organization-specific QA levels
    • Each level may have different reviewers and requirements
    • QA levels vary by site configuration
    • Chart cannot advance if QA flags require resolution
  3. Chart Demotion Process
    • When changes are requested, chart returns to S0 (Crew Level)
    • Original crew receives notification of required changes
    • Crew must address all QA flags before resubmission
    • Chart re-enters QA process after corrections

QA Flag Types & Categories

Clinical Care Flags

Purpose: Address clinical assessment, treatment, and patient care issues

  • Assessment Concerns: Physical exam or vital signs questions
  • Treatment Decisions: Medication or intervention choices
  • Protocol Adherence: Clinical protocol compliance
  • Patient Safety: Care quality and safety issues

Documentation Flags

Purpose: Address charting accuracy, completeness, and quality

  • Missing Information: Required fields or data not completed
  • Accuracy Issues: Incorrect times, data, or entries
  • Narrative Quality: LCHART/DRAATT format compliance
  • Attachment Requirements: Missing trends sheets or forms

Administrative Flags

Purpose: Address operational and administrative requirements

  • Process Compliance: Following organizational procedures
  • Training Requirements: Educational needs identified
  • Equipment Issues: Device or supply concerns
  • Communication: Notification and reporting requirements

Crew Feedback Flags

Purpose: Educational communication and discussion

  • IM-Based Communication: Does not create formal QA flag
  • Educational Purpose: Teaching and learning opportunities
  • Two-Way Discussion: Allows reply and conversation
  • PRID-Linked: Directly connected to specific chart

QA Flag Response Types

Required Response Types

When responding to QA flags, you must select the appropriate response type:

  • Acknowledgement: Most common response - confirms understanding and acceptance
  • Special Report: When additional incident reporting is required
  • Email Discussion: For detailed explanations requiring formal email response
  • Instant Message: For immediate clarification or quick communication
  • Addendum: Rarely used - for chart additions or corrections

Response Type Selection Guidelines

Acknowledgement

Use When: Standard flag response acknowledging the feedback or correction

  • Accepting constructive feedback
  • Confirming understanding of educational points
  • Simple agreement with suggested changes
  • Most routine QA flag responses

Example: "Acknowledged. Will ensure more detailed pain assessment documentation in future calls."

Special Report

Use When: QA flag identifies need for formal incident reporting

  • Patient safety events requiring documentation
  • Equipment malfunctions or failures
  • Protocol deviations needing formal review
  • Adverse events or complications

Example: "Special report initiated for medication dosage discrepancy requiring review."

Email Discussion

Use When: Detailed explanation or formal communication is needed

  • Complex clinical decision explanations
  • Disagreement requiring detailed justification
  • Educational discussions needing documentation
  • Multi-party communication requirements

Example: "Requesting email discussion to explain clinical reasoning for treatment protocol modification."

Instant Message

Use When: Quick clarification or immediate response is appropriate

  • Simple questions needing quick answers
  • Timeline clarifications
  • Minor documentation questions
  • Informal educational exchanges

Example: "Quick question about the time discrepancy - can we discuss via IM?"

Addendum Response: This response type is rarely used and should only be selected when making actual additions or corrections to the patient care record. Consult with supervisors before using.

Critical Response Deadlines

  • Next Day Response: All flags answered by 1400 (2:00 PM) if on shift
  • End-of-Shift Requirement: All flags answered before departing shift
  • Email Notifications: Automatic alerts sent when flags created
  • Response Tracking: System tracks who responded and how
  • Disciplinary Action: Failure to meet deadlines may result in discipline

Response Process

  1. Flag Notification
    • Email notification sent when QA flag created
    • Direct assign flags: Only assigned crew member notified
    • All crew flags: All crew members receive notification
    • Users can opt out of email notifications in preferences
  2. Flag Review & Response
    • Access flagged charts through Patient Records
    • Review QA flag details and requirements
    • Determine appropriate response type
    • Document response thoroughly and professionally
  3. Response Submission
    • Submit response through EMSCharts interface
    • Response becomes part of permanent chart record
    • QA flag creator receives notification of response
    • Flag marked as "handled" in system

X Series Monitor Integration & Data Import

Device Integration: The ZOLL X Series Advanced monitor can transmit patient data directly to EMSCharts through cloud connectivity, eliminating manual data entry and ensuring accuracy.

Connecting Personal Device to X Series Advanced

  1. Enable WiFi Hotspot on X Series Monitor
    • Navigate to Main Menu → Settings → Communications
    • Select "WiFi Hotspot" and enable the feature
    • Note the Network Name (SSID) and Password displayed
    • Ensure hotspot is broadcasting (indicator light will show)
  2. Connect Personal Device to Monitor Hotspot
    • Open WiFi settings on your smartphone/tablet
    • Locate the X Series network name (typically "XSeries_[Serial#]")
    • Enter the password displayed on the monitor screen
    • Verify connection established (connected icon appears)
  3. Access Monitor Web Interface
    • Open web browser on connected device
    • Navigate to monitor's IP address (typically 192.168.1.1)
    • Login with device credentials if prompted
    • Verify access to patient data and case information

Transmitting Call Data to Cloud

  1. Ensure Cloud Connectivity
    • Verify X Series has active cellular or WiFi connection
    • Check cloud sync status indicator (cloud icon)
    • Confirm ZOLL Cloud Services are configured
    • Validate agency cloud account is active
  2. Initiate Data Transmission
    • Complete patient case on X Series monitor
    • Navigate to Case Summary screen
    • Select "Transmit Case" or "Send to Cloud" option
    • Confirm patient information is accurate
    • Press "Transmit" to begin upload
  3. Monitor Transmission Status
    • Progress bar shows upload percentage
    • Wait for "Transmission Complete" confirmation
    • Note the Case ID for reference
    • Verify successful upload icon appears
Transmission Best Practice: Always transmit data while still on scene or immediately upon arrival at the hospital to ensure reliable connectivity. Delayed transmissions may fail if the monitor loses network connection.

Importing Data into EMSCharts

  1. Access EMSCharts Import Function
    • Log into EMSCharts web platform
    • Navigate to the patient's chart or create new chart
    • Click "Import Data" or "Device Import" button
    • Select "ZOLL X Series" from device list
  2. Locate Transmitted Case
    • System displays available cases from cloud
    • Search by date/time, patient name, or Case ID
    • Verify correct case by reviewing patient demographics
    • Select the appropriate case for import
  3. Review and Import Data
    • Preview imported data before accepting
    • Verify vital signs, timestamps, and interventions
    • Check ECG strips and waveform data
    • Click "Import" to populate chart fields
    • System maps data to appropriate chart sections
  4. Verify Imported Information
    • Review all imported vital signs for accuracy
    • Confirm medications and procedures imported correctly
    • Check that timestamps align with your documentation
    • Add any missing information manually
    • Document any discrepancies in narrative
Data Import Verification: Always review imported data for accuracy. While device integration reduces errors, you remain responsible for ensuring all documentation is complete and accurate before chart submission.

  • Password Issues: Verify correct password entry
  • Browser Errors: Clear cache and cookies

Transmission Failures

  • Upload Stuck: Check network connectivity
  • Partial Upload: Retry transmission
  • Cloud Errors: Verify cloud account status
  • Timeout Issues: Transmit smaller data sets

Import Problems

  • Case Not Found: Allow time for cloud sync
  • Data Mismatch: Verify correct patient selected
  • Missing Data: Check monitor case completion
  • Mapping Errors: Contact IT support

Data Quality Issues

  • Duplicate Vitals: Remove manual entries
  • Time Conflicts: Adjust to match narrative
  • Missing Interventions: Add manually
  • ECG Quality: Re-transmit if poor quality
