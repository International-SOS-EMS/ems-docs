Critical Elements to Verify

  • Complete dispatch number matches records HIGH
  • Separate chart for each patient (no combined charts)
  • Patient employer information documented (V2X, subcontractor, or military branch)
  • Correct base assignment and unit number
  • All crew members listed with roles and certifications

  • Emergency Calls: Must use actual dispatch terminology (NOT "Adult/Pediatric Medical/Trauma/Neuro")
  • ALL Transfers: Must be classified as "Transfer/Interfacility/Palliative Care" regardless of medical issues
  • Standby Events: Format as "Standby - [Event Type]"
  • Special Events: Format as "Special Event - [Description]"

üë§ Patient Information & Demographics Review

Required Classifications

Patient Must Be Classified As One Of:

  • CTR: Contractor personnel
  • DoD: Department of Defense civilian
  • Mil: Military personnel
  • CIV: Civilian (non-affiliated)

Address Documentation Standards

On Base Addresses

  • City: Use base name (Arifjan, Buehring)
  • Format: Zone, Street, Building, or Location
  • Country: Kuwait or Qatar
  • State: Leave blank

Off Base Addresses

  • Street: Include road/highway name
  • Reference: Cross street or landmark
  • Country: Kuwait or Qatar
  • GPS: Include if available

üè• Clinical Assessment Pages (2-5) Review

Page 2 Requirements

  • Impression/Diagnosis area completed HIGH
  • Chief complaint includes duration
  • ALS assessment documented with rationale
  • HPI includes mechanism of injury (if applicable)
  • Exposure documented ONLY if actual exposure occurred

Common Page 2 Errors

  • Missing pain assessment details
  • Incomplete medication/allergy documentation
  • HPI doesn't match chief complaint
  • Missing patient belongings documentation (especially CAC cards)

Page 3-5 Assessment Integrity

‚ö†Ô∏è Critical Review Point

Verify assessments match what was actually performed. Look for:

  • Default values left unchanged when assessment wasn't done
  • "Not done" option should be used when appropriate
  • GCS components must add up correctly
  • Airway status: Patent unless compromised/intubated
  • Pupils and motor/sensory MUST be documented

üìä Page 8: Activity Log & Vital Signs Review

‚ö†Ô∏è Mandatory Requirements

  • Minimum 2 sets of vital signs (BP, HR, RR, SpO2)
  • Vital signs at appropriate intervals based on condition
  • Final set within 5 minutes of handoff
  • HR and BP methods must be listed

Special Monitoring Requirements

Condition Required Monitoring Flag if Missing
Abdominal Pain Cardiac monitor per protocol Yes - Protocol violation
Narcotic Administration 4-lead + ETCO2 required Yes - Critical safety issue
Recent Narcotic Use (within 30 min) 4-lead + ETCO2 required Yes - Critical safety issue
Pain Medication Given Pain scale before and 10 min after Yes - Documentation requirement

üìÖ Timeline Consistency Check

Verify times match across:

  • Page 1 dispatch times
  • Page 8 vital sign times
  • Activity log entries

Inconsistent times = Immediate flag

Trends Sheet Requirements

Mandatory Upload Verification

  • File Name: Must be "PRID#_Trends" (e.g., 2024-12345_Trends)
  • Format: PDF scan only (no photos with backgrounds)
  • Category: Must use TRENDS category
  • Content: Must match documented vital signs

üìù Narrative Documentation Review

‚ö†Ô∏è Mandatory Format Requirement

ALL narratives MUST use either LCHART or DRAATT format

Implementation Date: March 1, 2025

Format Verification

LCHART Components

  • L - Location documented
  • C - Chief Complaint in patient's words
  • H - History complete
  • A - Assessment findings
  • R - Rx (treatments) listed
  • T - Transport details

DRAATT Components

  • D - Dispatch information
  • R - Response details
  • A - Assessment findings
  • A - Actions taken
  • T - Transport information
  • T - Transfer details

Common Narrative Errors to Flag

  • Missing components from chosen format
  • Timeline doesn't match other sections
  • Ambiguous language (e.g., "patient felt better" vs specific improvement)
  • Missing pertinent negatives
  • Incomplete vital signs documentation

  • Treatment decisions
  • Protocol adherence
  • Patient safety issues

Documentation

  • Missing information
  • Accuracy issues
  • Narrative quality
  • Missing attachments

Administrative

  • Process compliance
  • Training requirements
  • Equipment issues
  • Communication

Flag Comment Best Practices

‚úì Effective Flag Comments Include:

  • Specific Issue: Clearly state what needs correction
  • Location: Reference page/section where issue occurs
  • Required Action: What needs to be done to resolve
  • Educational Tone: Focus on improvement, not criticism

Example: "Page 8: Vital signs missing at handoff. Please add final set of vitals taken within 5 minutes of patient transfer as per protocol."
